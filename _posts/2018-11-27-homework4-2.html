A. Refine your project proposal.

Objective:

요즘 SNS를 사용하는 많은 사람들은 음식점에 들어가면 젓가락을 들기보다 핸드폰을 먼저 들어 사진을 찍습니다. 하지만 정작 나중에 사진들이 쌓이다 보면 자신이 어떤 음식점을 갔는지 기억도 잘 나지 않고, 그 사진을 찾고싶어도 찾기가 힘듭니다. 본 application은 사용자들이 자신이 방문한 음식점의 음식 사진들을 찍어서 업로드 함으로써 자신의 음식 사진들을 쉽게 관리해주고, 다른 사람들이 업로드한 사진들을 통해 음식점 추천도 받을 수 있는 application 입니다. 그렇기에 다양한 기능을 넣기보다는 음식사진을 저장해두고 다른사람들의 사진을 볼 수 있는 기능에만 초점을 두었습니다.

Functional requirements


1. 사용자가 원하는 사진들을 선택하고 개별적으로 쉽게 보정하고 업로드 할 수 있다.
2. 사진 업로드시 간단한 평가와 사진만 있으면 되기 때문에, 다른사람들이 원하는 사진을 찾을 때 용이하다.
3. 사진을 검색할 떄 음식메뉴, 지역(시/구/동) 단위로 검색할 수 있다.
4. AI를 통한 음식점 추천을 받을 수 있다.
- 사용자가 많이 올린 음식 카테고리 범주안에서 추천 - 사용자가 현재 있는 위치나, 선택한 지역에서 인기있는 음식점을 추천 - 시간대(점심, 저녁)나, 날씨(비올때)를 고려하여 추천


UI requirements


1. 기본적으로 같은 음식점끼리 그룹화되어 사진들을 보여줌으로써, 여러 사람들이 찍은 같은 메뉴의 사진들을 한눈에 보기 쉽다.
2. 사용자가 자신이 올린 사진들만 볼 수 있으며, 한눈에 보기 쉽게 사진들만 볼 수도 있고, 자신이 작성한 정보들(메뉴, 음식점, 장소)을 함께 볼 수도 있다.
3. 사진들을 메뉴, 지역을 설정해 두고 검색할 때에도 사람들이 업로드한 사진의 갯수가 많은 순으로 음식점들의 사진들을 보여준다.
4. 음식 메뉴는 없을 시에 사용자가 직접 추가 할 수 있고, 한 사용자가 추가하면 다른 모든 사용자도 그 메뉴를 선택할 수 있게 된다.
5. 누구나 사용하기 쉽도록 복잡한 기능 없이 업로드와 사진들을 볼 수 있는 기능만 제공한다.
6. 사진이 돋보이도록 배경색은 흰색을 사용한다.

Scenarios I


1. 음식점에서 먹음직스러운 음식이 나와서 사진을 찍는다.
2. Yam-Yam(본 application)에서 방금 찍은 사진을 업로드 하겠냐는 팝업창이 표시된다.
3. 팝업창을 터치해서 application 을 실행시키면 자동적으로 방금 찍은 사진을 업로드 하는 창으로 들어가게 된다.
4. 사진을 보정, 편집을 하고 정보(메뉴, 음식점, 위치, 맛평가) 들을 입력한다.
5. '완료' 버튼을 통해 업로드를 완료한다.
6. Yam-Yam 을 종료한다.


Scenarios II


1. 한 사용자가 저녁에 친구와 곱창을 먹기로 했지만 어떤 음식점을 가야할지 고민하고 있다.
2. Yam-Yam 을 실행한다.
3. 음식 카테고리에서 곱창을 선택하고, 지역을 선택한다.
4. 다른 사용자들이 해당 지역에서 찍고 업로드한 곱창 사진들이 음식점 별로 나열된다.
5. 사용자는 사진들을 오늘 어떤 음식점에 갈 지 선택할 수 있다.
6. Yam-Yam 을 종료한다.


Task Model I : 새로운 사진 업로드


1. 아이콘을 클릭해 application 을 실행.
2. '업로드' 버튼을 클릭.
3. 내 사진첩이 열리고 그 중 업로드를 원하는 사진들을 선택.
4. '다음' 버튼 클릭.
5. 각 사진별로 편집/보정.
6. '다음' 버튼 클릭.
7. 각 사진들의 정보를 입력.
8. '완료' 버튼 클릭.


Task Model II : 내 주변 맛있는 곱창 집 검색하기


1. 아이콘을 클릭해 application 을 실행.
2. '검색' 버튼을 클릭.
3. '메뉴' 에서 '곱창'을 검색 하여 선택.
4. '내 위치 중심' 버튼 클릭.
5. '검색하기' 버튼 클릭.


B. Design the interface.


1. 선호도 조사

신규 유저는 초기에 유저에 대한 정보가 없기 때문에 메뉴 추천을 위한 정보 수집을 한다. 사용자가 알아보기 쉽게 큰 원형 사진으로 버튼을 구성하여 선택하기 쉽게 한다.


2. 메인화면

바탕화면은 사진을 돋보이게 해주는 흰색,
제목과 평점은 흰배경과 검은 글씨들 사이에서 잘 보일 수 있도록 빨강을 선택하였다. 사진들은 기본적으로 사람들이 보기 편하게 같은 음식점 사진은 한 줄에 나열되어 옆으로 넘기며 볼 수 있다.
큰 메뉴(홈, 업로드, 내 사진목록)은 한손으로 선택하기 편하게 아래쪽으로 배치했고, 설정과 검색 버튼은 위쪽에 배치하였다.


3. 설정 메뉴

설정 메뉴의 아이콘들은 직관적으로 알아보기 쉬운 아이콘을 사용하여 표시하였다.
