A. Refine your project proposal.

Project:

사용자들이 자신이 방문한 음식점의 음식 사진들을 찍어서 업로드 함으로써 자신의 음식 사진들을 쉽게 관리해주고, 다른 사람들이 업로드한 사진들을 통해 음식점 추천을 받을 수 있는 application 이다.

Platform : Android App

대부분의 사람들이 핸드폰으로 사진을 찍기 때문에 웹 보다는 핸드폰으로 사용가능한 어플리케이션이어야 업로드를 포함한 대부분의 사용이 편리하다는 의견에 따라 변경하였다.


Functional requirements


1. 자신의 사진을 업로드, 삭제 하는 기능.
2. 업로드 할 사진을 편집하는 기능.
3. 지역, 메뉴, 음식점 별로 사진을 정렬해주는 기능.
4. AI를 통해 추천 음식점을 제공해주는 기능


UI requirements


1. 메인화면 : 다른 모든 사용자들이 당일에 올린 사진들을 보여준다.
2. 사진 업로드 버튼을 누르게 되면 내 핸드폰에 저장되어있는 사진들을 보여주고, 그 중 원하는 사진들을 선택해서 업로드 할 수 있다.
3. 사진들을 지역, 메뉴, 음식점 별로 볼 수 있는 정렬 필터가 존재한다.
4. 음식 메뉴의 카테고리는 사용자가 직접 추가 할 수 있고, 한명이 추가하면 다른 모든 사용자도 그 카테고리를 보고 사용할 수 있게 된다.
5. 복잡하지 않고 직관적인 UI를 제공한다.
6. 음식이 돋보이도록 기본 배경색은 흰색을 사용한다.

Scenarios I


1. 음식점에서 먹음직스러운 음식이 나와서 사진을 찍는다.
2. Yam-Yam(본 application)에서 방금 찍은 사진을 업로드 하겠냐는 팝업창이 표시된다.
3. 팝업창을 터치해서 application 을 실행시키면 자동적으로 방금 찍은 사진을 업로드 하는 창으로 들어가게 된다.
4. 사진을 보정, 편집을 하고 정보(메뉴, 음식점, 위치, 맛평가) 들을 입력한다.
5. '완료' 버튼을 통해 업로드를 완료한다.
6. Yam-Yam 을 종료한다.


Scenarios II


1. 한 사용자가 저녁에 친구와 곱창을 먹기로 했지만 어떤 음식점을 가야할지 고민하고 있다.
2. Yam-Yam 을 실행한다.
3. 음식 카테고리에서 곱창을 선택하고, 지역을 선택한다.
4. 다른 사용자들이 해당 지역에서 찍고 업로드한 곱창 사진들이 음식점 별로 나열된다.
5. 사용자는 사진들을 오늘 어떤 음식점에 갈 지 선택할 수 있다.
6. Yam-Yam 을 종료한다.


Task Model I : 새로운 사진 업로드


1. 아이콘을 클릭해 application 을 실행.
2. '업로드' 버튼을 클릭.
3. 내 사진첩이 열리고 그 중 업로드를 원하는 사진들을 선택.
4. '다음' 버튼 클릭.
5. 각 사진별로 편집/보정.
6. '다음' 버튼 클릭.
7. 각 사진들의 정보를 입력.
8. '완료' 버튼 클릭.


Task Model II : 내 주변 맛있는 곱창 집 검색하기


1. 아이콘을 클릭해 application 을 실행.
2. '검색' 버튼을 클릭.
3. '메뉴' 에서 '곱창'을 검색 하여 선택.
4. '내 위치 중심' 버튼 클릭.
5. '검색하기' 버튼 클릭.
